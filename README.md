## 视角

用户视角（能不能解决我的问题）：

- 是什么：这东西到底是干嘛的？能解决我什么痛点？
- 怎么用：界面友好吗？要不要学半天？手机/电脑都能用？
- 值不值：免费？试用？付费多少钱？性价比高吗？
- 安全吗：会偷我数据吗？账号安全吗？

产品经理（能不能规模化解决一类人的问题）：

- 用户价值与场景：标用户是谁？解决什么场景下的核心痛点？相比现有方案的差异化点（Moat）在哪里？
- 产品逻辑与体验：信息架构是否合理？核心路径是否<3步？新手引导/空状态/异常处理是否完善？
- 数据指标定义：DAU/留存/转化漏斗怎么设计？AARRR各环节的North Star Metric是什么？
- 竞品与迭代：竞品矩阵（功能/定价/用户评价）；MVP验证假设 → 迭代路线图（Roadmap）
- 商业闭环：变现模式（订阅/广告/企业版）？LTV vs CAC 是否健康？
- 设计语言与程序价值。

决策路径

用户视角（好奇 → 试用 → 口碑 → 付费/放弃）
产品经理（用户痛点 → MVP假设 → 数据验证 → 迭代优先级 → 商业模型）
运营人员（冷启动 → 拉新 → 激活 → 留存 → 营收 → 防刷/防作弊）
市场营销（人群洞察 → 创意素材 → 渠道投放 → 数据归因 → 优化迭代）
程序员（技术选型 → 原型验证 → 压力测试 → 灰度发布 → 监控优化）
